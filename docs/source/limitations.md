# Current Limitations

## Packaging

The active prototype repo is not yet the polished package interface that end users should rely on long term.

## Testing

The documented code path has compile validation, but live provider behavior depends on real credentials and network access.

## API stability

Class names and method shapes are still simple and may change as the package design stabilizes.

## Error handling

Provider and network failures are surfaced, but the exception model is still lightweight.

## Tooling

`tools.py` is currently a placeholder.
